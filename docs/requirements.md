# AI 资讯站 Requirements

版本：v0.1  
日期：2026-07-06  
阶段：Requirements  
状态：已定稿，可进入 Design 阶段

## 1. 产品目标

建设一个面向 AI 资讯关注者和 AI 工具学习者的 Web 应用。用户每天可以快速浏览全球 AI 新闻，也可以系统阅读 AI 工具使用技巧和方法文章。

产品第一版不做社区讨论，不做评论区，不做复杂推荐系统。第一版重点是可信内容、清晰来源、用户收藏、点赞、作者主页和可持续扩展的内容结构。

## 2. 核心用户

### 2.1 每日 AI 新闻读者

需求：

- 快速知道全球 AI 行业发生了什么。
- 判断信息来源是否可靠。
- 收藏重要新闻，稍后阅读。
- 通过热榜发现高价值内容。

### 2.2 AI 工具学习者

需求：

- 学习具体 AI 工具的使用技巧。
- 查找可复用的方法、案例和步骤。
- 收藏教程，形成自己的学习资料库。
- 关注持续产出高质量教程的作者。

### 2.3 注册作者

需求：

- 发布原创文章、工具教程或整理型内容。
- 通过个人主页展示自己的文章。
- 查看自己的内容获得了多少点赞、收藏和关注。

## 3. 内容板块

### 3.1 全球 AI 新闻资讯

板块定位：快速、可信、可追溯。

允许内容类型：

- 原创新闻报道。
- 外部新闻摘要和导读。
- 外部文章编译/整理。
- 发布会、产品更新、融资、模型发布、政策监管等资讯。

必填字段：

- 标题
- 摘要
- 正文或导读
- 发布时间
- 内容类型：`original` / `repost_summary` / `curated`
- 来源名称
- 来源 URL
- 作者或整理者
- 标签

规则：

- 如果内容来自外部来源，必须显示原文出处。
- 如果不是原创，详情页必须在标题附近显示来源名称和原文链接。
- 外部来源不得伪装成本站原创。

### 3.2 AI 工具使用技巧和方法文章

板块定位：可学习、可操作、可收藏。

允许内容类型：

- 单工具教程。
- 多工具组合工作流。
- 提示词/自动化/插件使用方法。
- 案例复盘。
- 新手入门指南。
- 进阶技巧。

必填字段：

- 标题
- 摘要
- 正文
- 作者
- 适用工具名称
- 工具官网或参考链接
- 难度：`beginner` / `intermediate` / `advanced`
- 预计阅读时间
- 适用场景
- 核心步骤
- 标签

可选字段：

- 所需账号或付费计划
- 支持平台：Web / iOS / Android / macOS / Windows / API
- 示例提示词
- 示例输出
- 注意事项
- 替代工具

## 4. 投稿审核规则

### 4.1 投稿权限

注册用户可以提交文章，但默认不能直接公开发布。

默认发布流程：

1. 注册用户创建文章草稿。
2. 用户提交审核。
3. 管理员或编辑审核。
4. 审核通过后文章公开。
5. 审核拒绝后退回草稿，并显示拒绝原因。

### 4.2 可信作者机制

第一版预留 `trusted_author` 标记，但不必做复杂后台。

规则：

- 普通作者：提交后进入 `pending_review`。
- 可信作者：可以直接发布，但管理员可以下架。
- 管理员：可以创建、编辑、发布、下架所有内容。

### 4.3 文章状态

文章状态必须包含：

- `draft`：草稿。
- `pending_review`：待审核。
- `published`：已发布。
- `rejected`：审核拒绝。
- `archived`：作者归档或管理员下架。

前台公开列表只展示 `published` 状态文章。

## 5. 转载与版权边界

### 5.1 默认策略

本站默认不允许完整搬运外部付费或受版权保护文章。外部新闻内容默认以摘要、导读、评论、编译或链接跳转形式呈现。

### 5.2 转载内容展示规则

转发或整理外部资讯时，必须展示：

- 来源名称。
- 原文链接。
- 原文发布时间，如果可获得。
- 本站整理/编译者。
- 内容类型标识：转载摘要 / 编译整理 / 原创。

### 5.3 外部正文使用边界

允许：

- 用自己的话总结新闻事实。
- 引用必要的短句，并标明来源。
- 链接到原文。
- 对外部信息进行分析、归纳、对比和评论。

不允许：

- 未授权全文转载。
- 删除来源后伪装原创。
- 大段复制外部文章。
- 转载明显禁止转载的内容。
- 把付费墙内容完整搬运到本站。

### 5.4 投稿者责任确认

用户提交非原创内容时，发布表单必须要求确认：

- 我已填写原文出处。
- 我确认该内容不是未授权全文转载。
- 我确认标题、摘要和正文没有伪装原创。

## 6. 管理员角色

第一版必须有管理员角色，但管理后台可以先做最小能力。

### 6.1 角色

- `guest`：未登录用户。
- `user`：注册用户。
- `trusted_author`：可信作者。
- `editor`：编辑，可审核内容。
- `admin`：管理员，拥有全部管理权限。

### 6.2 权限

guest：

- 浏览公开文章。
- 浏览用户公开主页。
- 搜索和筛选公开内容。
- 不能点赞、收藏、关注、投稿。

user：

- 点赞。
- 收藏。
- 关注作者。
- 创建草稿。
- 提交审核。
- 编辑自己的未发布文章。
- 查看自己的文章状态。

trusted_author：

- 拥有 user 权限。
- 可直接发布自己的文章。

editor：

- 审核文章。
- 通过或拒绝投稿。
- 修改内容分类和标签。
- 下架违规文章。

admin：

- 管理用户角色。
- 管理所有文章。
- 管理标签。
- 管理首页置顶内容。
- 查看基础内容数据。

## 7. 首页排序规则

首页目标：兼顾时效、可信度和用户兴趣，不做纯热度榜。

### 7.1 首页区域

首页包含：

- 今日重点：编辑置顶或系统精选。
- 最新 AI 新闻：按发布时间排序。
- 热门工具技巧：按热度排序。
- 全站热榜：新闻和技巧混合排序。
- 标签入口：模型、产品、工具、教程、政策、融资等。

### 7.2 今日重点排序

规则：

1. 管理员或编辑置顶优先。
2. 没有置顶时，取最近 48 小时内热度最高的新闻。
3. 同分时发布时间更新者优先。

### 7.3 最新 AI 新闻排序

规则：

1. 仅展示 `section = news`。
2. 按 `published_at` 从新到旧排序。
3. 同发布时间时，热度更高者优先。

### 7.4 热门工具技巧排序

规则：

1. 仅展示 `section = tips`。
2. 默认统计最近 30 天。
3. 按热度分从高到低排序。
4. 同分时收藏数更多者优先。

### 7.5 全站热榜排序

第一版热度公式：

`score = likes_count + bookmarks_count * 2`

原因：

- 收藏比点赞更能代表长期价值。
- 第一版不引入复杂推荐算法，避免黑箱排序。

热榜时间窗：

- 默认展示最近 7 天。
- 保留切换：24 小时 / 7 天 / 30 天。

同分排序：

1. 收藏数更多者优先。
2. 点赞数更多者优先。
3. 发布时间更新者优先。

## 8. 点赞、收藏、关注

### 8.1 点赞

规则：

- 登录用户可点赞文章。
- 每个用户对同一文章最多点赞一次。
- 再次点击取消点赞。
- 未登录用户点击点赞时跳转或弹出登录引导。

### 8.2 收藏

规则：

- 登录用户可收藏文章。
- 每个用户对同一文章最多收藏一次。
- 再次点击取消收藏。
- 收藏文章进入个人收藏列表。

### 8.3 关注作者

规则：

- 登录用户可关注作者。
- 用户不能关注自己。
- 再次点击取消关注。
- 用户主页展示被关注数。

## 9. 用户个人子页面

每个注册用户自动生成公开子页面。

URL 规则：

- `/u/{username}`

页面展示：

- 用户头像。
- 显示名。
- 简介。
- 被关注数。
- 文章数量。
- 获赞总数。
- 被收藏总数。
- 用户发布的所有公开文章。

文章排序：

`likes_count + bookmarks_count` 从高到低排序。

同分排序：

1. 收藏数更多者优先。
2. 点赞数更多者优先。
3. 发布时间更新者优先。

不展示：

- 评论区。
- 访客留言。
- 私信入口。

## 10. 搜索与筛选

第一版需要基础搜索，不做语义搜索。

搜索范围：

- 标题。
- 摘要。
- 标签。
- 工具名称。
- 来源名称。
- 作者显示名。

筛选项：

- 内容板块：新闻 / 技巧。
- 标签。
- 难度，仅技巧文章。
- 发布时间。
- 来源类型。

默认排序：

- 搜索结果默认按相关度。
- 用户可切换为最新或热度。

## 11. 最小页面范围

MVP 必须包含：

1. 首页。
2. 全球 AI 新闻列表页。
3. AI 工具技巧列表页。
4. 文章详情页。
5. 登录页。
6. 注册页。
7. 用户个人子页面。
8. 个人收藏页。
9. 投稿/编辑文章页。
10. 最小审核管理页。

## 12. MVP 不做范围

第一版明确不做：

- 评论。
- 评分。
- 私信。
- 复杂社交动态流。
- 付费订阅。
- AI 自动生成整篇新闻。
- 个性化推荐算法。
- 多语言站点。
- 移动 App。
- 完整 CMS 后台。

约束：

- email 唯一。

### 13.2 articles

必需字段：

- id
- title
- slug
- summary
- body
- cover_image_url
- section
- status
- author_user_id
- source_type
- source_name
- source_url
- original_published_at
- published_at
- rejected_reason
- created_at
- updated_at

约束：

- slug 唯一。
- section 只能是 `news` 或 `tips`。
- status 使用第 4.3 节枚举。
- source_type 使用 `original` / `repost_summary` / `curated`。
- 非原创内容必须填写 source_name 和 source_url。

### 13.3 article_tip_metadata

仅技巧文章需要。

字段：

- article_id
- tool_name
- tool_url
- difficulty
- estimated_reading_minutes
- use_case
- required_plan
- platforms
- example_prompt
- example_output
- caveats

### 13.4 interactions

需要：

- article_likes
- article_bookmarks
- follows

约束：

- `article_likes(user_id, article_id)` 唯一。
- `article_bookmarks(user_id, article_id)` 唯一。
- `follows(follower_user_id, followed_user_id)` 唯一。

### 13.5 moderation_logs

字段：

- id
- article_id
- reviewer_user_id
- action
- reason

- `approved`
- `rejected`

## 14. 功能验收标准

### 14.1 注册登录

- 用户可以注册、登录、退出。
- 重复邮箱不能注册。
- 重复用户名不能注册。
- 未登录用户不能点赞、收藏、关注、投稿。

### 14.2 投稿审核

- 普通用户提交文章后进入待审核。
- 待审核文章不出现在公开列表。
- 编辑可以通过或拒绝文章。
- 拒绝文章必须有原因。
- 通过后文章进入公开列表。

### 14.3 转载来源

- 非原创文章未填写 source_name 或 source_url 时不能提交审核。
- 非原创文章详情页必须显示来源。
- 原创文章显示作者署名。

### 14.4 点赞收藏

- 同一用户不能重复点赞同一文章。
- 同一用户不能重复收藏同一文章。
- 点赞和收藏可以取消。
- 文章计数实时或准实时更新。

### 14.5 用户主页

- 每个用户有 `/u/{username}` 页面。
- 主页展示被关注数、文章数量、获赞总数、被收藏总数。
- 主页文章按点赞数和收藏数总和从高到低排序。
- 只展示该用户已发布文章。

### 14.6 首页排序

- 最新新闻按发布时间倒序。
- 热门工具技巧按热度排序。
- 全站热榜使用 `likes_count + bookmarks_count * 2`。
- 编辑置顶内容优先展示在今日重点。

## 15. 非功能要求

### 15.1 可访问性

- 所有按钮和链接必须支持键盘访问。
- 表单必须有 label。
- 焦点状态必须可见。
- 正文移动端字号不小于 16px。
- 颜色对比满足 WCAG AA。

### 15.2 性能

- 首页首屏内容应优先加载。
- 文章列表分页或无限加载二选一，MVP 推荐分页。
- 图片必须有尺寸约束和懒加载。

### 15.3 SEO

- 文章详情页必须有 title、description。
- 公开文章应有可读 slug。
- 非原创文章 canonical 规则在设计阶段决定，MVP 至少保留原文链接。

### 15.4 安全

- 密码必须哈希存储。
- 投稿正文必须防 XSS。
- 管理员接口必须鉴权。
- 用户只能编辑自己的文章，管理员除外。

## 17. 当前决策摘要

- 投稿审核：普通用户投稿必须审核；可信作者可直发。
- 转载版权：默认只允许摘要、导读、编译和链接，不允许未授权全文转载。
- 管理员角色：MVP 必须有 admin/editor/trusted_author/user/guest。
- 首页排序：编辑置顶 + 最新 + 热榜混合，热榜公式为 `likes_count + bookmarks_count * 2`。
- 工具文章字段：必须有工具名、工具链接、难度、阅读时间、适用场景、核心步骤。
